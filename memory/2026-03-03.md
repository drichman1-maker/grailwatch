## 5-Part Prompting Framework (for Parallel Work)

Use this structure when spawning sub-agents or requesting complex tasks:

**1. Persona** — Who to be
- *Example: "You are a meticulous code reviewer"*

**2. Context** — Background info  
- *Example: "This is a React app using Vite, deployed on Vercel"*

**3. Task** — What to do specifically
- *Example: "Refactor the ProductDetail component to use TypeScript"*

**4. Format** — How to present output
- *Example: "Give me the full file with inline comments explaining changes"*

**5. Constraints** — Limitations/rules
- *Example: "Don't change the API calls, only the component structure"*

**Full Example:**
> "You are a senior React developer (1). This is a price tracking app with Recharts for visualization (2). Create a PriceHistoryChart component that shows 90 days of price data (3). Output the full JSX file with PropTypes (4). Don't install new dependencies, use what's already in the project (5)."
